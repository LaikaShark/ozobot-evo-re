  python3 ozobot.py flash examples/tests/t00_led_red.ff
      fill terminal bg (works on WSL/headless); ENTER to flash

THE LED ORACLE -- HOW A TEST REPORTS PASS/FAIL
-------------------------------------------------------------------------------

Most tests compute something and then self-report with the LED:

  GREEN = PASS      the robot computed the expected result.
  RED   = FAIL      it computed the wrong result (a real bug).
  NONE  = no color  the program didn't run at all (reception failed, or the
                    robot rejected the program -- e.g. a wrong capability byte).

Oracle tests hold the color in an infinite loop, so it stays lit until you
re-flash or power-cycle -- no need to catch a brief flash. Action tests
(move/turn/audio) run once and stop.

Run Block 1 first. The oracle depends on `if/else`, `=`, and `led` being
correct; Block 1 proves those, so later GREEN/RED readings are trustworthy.

no test emits opcode 0xC7 Safe to flash.
